<comment id="14676940" author="JIRAUSER20910" created="Tue, 28 May 2024 13:41:07 -0700">[jdk17u-fix-request] <br/> This change fixes an issue caused by usrmerge on Debian-based systems - dpkg -S does not return a package containing a symlinked shared library in /lib, e.g. <br/> ``` <br/> dpkg -S /lib/x86_64-linux-gnu/libmd.so.0 <br/> # dpkg-query: no path found matching pattern /lib/x86_64-linux-gnu/libmd.so.0 <br/> ``` <br/> <br/> The fix calls toRealPath() to resolve an actual path to the file before dpkg call and falls back to the old behaviour if the dpkg call fails. <br/> <br/> The change is tested by jpackage set of jtreg tests and should be low risk.</comment>
